# Git / VCS 정책

## 자동 커밋·푸시 금지

- 코드를 수정한 뒤 **절대 자동으로 커밋하거나 푸시하지 않는다**.
- 사용자가 명시적으로 "커밋해줘" 또는 "푸시해줘"라고 요청했을 때만 실행한다.
- 코드 수정 승인 ≠ 커밋 승인.

## 커밋 메시지

- 한 줄 제목 + 빈 줄 + 본문(필요 시).
- 제목은 50자 이내, 한국어 또는 영어. 동사 원형으로 시작.
- 본문은 *왜* 변경했는지 위주.

## 분기 전략

- MVP 단계에선 단일 `main` 브랜치 사용.
- 큰 작업은 feature branch + PR 자율 결정.

## 금지

- `--no-verify` 로 훅 우회
- `git push --force` (사용자 요청 시에만)
- 시크릿 파일 커밋 (.env 등)
